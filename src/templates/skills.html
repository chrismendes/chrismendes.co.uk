Grunt

Automation tool that aids development, testing, and optimisation.

I use Grunt for JavaScript linting, compiling Sass/LESS, generating spritesheets, browser auto-reloading, and for other development admin.

But also for building and optimising an app fit for a production environment, involving code concatenation and minification for example.

A few of my favourite Grunt plugins are:

  • grunt-browser-sync - Watches code and injects changes into the browser on each target device at once, and keeping them all in sync when navigating
  • grunt-rev - Appends to asset filenames a unique content-based hash for anti-caching purposes
  • grunt-spritesmith - Generates sprite sheets and CSS based on specified image files

Charles

Web debugging, monitoring, and proxy tool.

I have benefited a lot from using Charles not only as a HTTP Monitor, for which there are many tools, but also as a means to develop apps with external dependencies.

As a passionate and determined web developer, I always have two or three major learning objectives that I work towards in my spare time. Currently they are:

Currently the JavaScript MVC framework that I am experienced with is Backbone.js. The reason for this is because Backbone was the first JS framework that I encountered and I really liked the instant MVC structure that it enforces.

However, AngularJS for a while now seems to have taken the lead at least in terms of popularity. But more importantly, the general wisdom seems to be that choosing a particular framework over another depends on the project at hand - rather than the result of any direct comparisons.

I would therefore like to further extend my resourcefulness and flexibility by adding AngularJS to my arsenal.

It is my intention in the very near future, as a learning exercise, to create a separate branch of this website built with AngularJS rather than Backbone.js.

As part of my overall objective of becoming a complete developer, or as close to this as it is possible at any one point in time, I would like to possess a mobile app development capability.

I think that it's obvious to say that a developer who can offer mobile apps, rather than just responsive websites, can instantly become double the asset that they were previously for any given company.

Furthermore, for the last several months I have been working on a concept, as well as UI wireframes and technical architecture, for an ambitious mobile app that I intend to build and sell as a product of a new business.

I have chosen PhoneGap as the platform that will box the project as a mobile app, and with the learning materials that I have acquired I will soon begin work on an early prototype.

Also on the radar:

Having experimented with writing CoffeeScript, I really appreciate the stripped back, simplified, and quick-to-read JavaScript that CoffeeScript brings about. As a learning exercise I'm keen to re-write this website using CoffeeScript and keep it on a separate branch.

Node.js is obviously a bit of a revolution in the JavaScript world with respect to more efficient server-side processing. Given its growing popularity and adoption in various industries, I intend to be able to provide a Node.js service in the future.

A very popular alternative database solution, MongoDB seems to have propelled the NoSQL movement forwards somewhat. As someone who has always met these requirements previously with PHP & MySQL, I would like to adapt and add a NoSQL string to my bow.
