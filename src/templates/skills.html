www.gruntjs.com

Grunt

Automation tool that aids development, testing, and optimisation.

I use Grunt for JavaScript linting, compiling Sass/LESS, generating spritesheets, browser auto-reloading, and for other development admin.

But also for building and optimising an app fit for a production environment, involving code concatenation and minification for example.

A few of my favourite Grunt plugins are:

  • grunt-browser-sync - Watches code and injects changes into the browser on each target device at once, and keeping them all in sync when navigating
  • grunt-rev - Appends to asset filenames a unique content-based hash for anti-caching purposes
  • grunt-spritesmith - Generates sprite sheets and CSS based on specified image files

charlesproxy.com

Charles

Web debugging, monitoring, and proxy tool.

I have benefited a lot from using Charles not only as a HTTP Monitor, for which there are many tools, but also as a means to develop apps with external dependencies.

For example, using Charles' proxy functionality on a recent project, I have been able to redirect requests to an external live video stream (.m3u8 file) to a local version of this file, in order to change the contents of it for experiment and testing purposes.

Furthermore, using the throttling feature it is very easy to test the performance of a site under a limited Internet connection.

developer.yahoo.com/yslow

YSlow

Analysis tool for measuring the level of optimisation.

Relevant to one of my key interests, web performance, I very often rely on YSlow as a standard to judge whether or not basic principles for good web performance has been implemented or not.

This helps in providing a checklist for developing my own sites, but also a means to assess others.

In addition, I often call upon other tools to help enhance web performance such as Google PageSpeed Insights and webpagetest.org.

virtualbox.org

VirtualBox

Tool for setting up a virtual environment.

My number one port of call when undertaking cross-browser testing especially in relation to Internet Explorer.

Downloading the necessary installation files from the modern.ie website, developers can test applications in any version of IE including as far back as IE6.

browserstack.com

BrowserStack

Online virtual environments for testing purposes.

I'm a big fan of BrowserStack as I appreciate the "all under one roof" approach in providing almost every browser and every mobile device to test with.

The big advantage is that it's much easier and quicker to start testing. The big drawback however is the subsequent speed of the service waiting for pages to render, which isn't always great.

But other features including the ability to acquire specially prepared screenshots to share with others is very useful, helping to provide assurance to other stakeholders.
